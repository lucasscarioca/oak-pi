name: shipout
description: Check out or create an appropriate branch, then commit and push the current work. Use only when the user explicitly invokes "$shipout" or directly asks to use the shipout skill; do not use proactively for general implementation or validation tasks.
---

Check out onto a new branch for this work, then commit and push it.

Do the following:
- Check out the appropriate branch for the work, creating one if needed.
- Make sure the changes are ready to ship.
- Create a clear commit message that follows Conventional Commits, including scope when it is obvious.
- Commit the changes.
- Push the branch.
